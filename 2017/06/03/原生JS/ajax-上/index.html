###AJAX
ajax即 Aasynchronous Javascript And XML(异步JavaScript和XML),是指一种创建交互式网页应用的网页开发技术，用于快速创建动态网页的技术；可以实现网页的异步更新。
它利用浏览器提供操作HTTP的接口（XMLHttpRequest或者ActiveXObject）来操作HTTP已达到异步效果。

###HTTP简介
HTTP：超文本传输协议。在ISO七层模型中属于应用层协议。
附：ISO七层（物理层、数据链路层、网络层、传输层、会话层、表示层、应用层 自而上底）

  • HTTP事务，一个HTTP事务由一条（从客户端发往服务器的）请求命令和一个（从服务器发往客户端的）响应结果组成。只有当请求和响应都成功时HTTP事务才算成功。

  • HTTP报文
    HTTP报文是由一行一行的简单字符串组成的。HTTP报文是纯文本，不是二进制代码，方便进行读写。HTTP报文分为三部分：
    1.起始行：报文的第一行就是起始行，在请求报文中用来说明做些什么；在响应报文中说明出现了什么情况。
    2.首部字段：起始行后面有零个或多个首部字段。每个首部字段都包含一个名字和一个值，首部分为5中类型：通用首部、请求首部、响应首部、实体首部、扩展首部
    3.主体：报文主体包括了所有类型的数据。请求主体中报错了要发送给web服务器的数据；响应主体中装载了要返回给客户端的数据。起始行和首部字段是结构化的文本形式，而主体可以包含任意的二进制数据。当然主体也可以包含文本。

    HTTP方法

    http支持几种不同的请求命令，这些命令被称为http方法（动词）。每条http请求报文都包括一个方法。这个方法会告诉服务器执行什么动作（获取一个web页面、删除一个文件、、）
    常见方法如下：

    • GET：主要是传给服务器一些参数来获取服务器上指定的资源
    • POST：将客户端数据发送到一个服务器网关程序
    • DELETE：从服务器上删除命名资源
    • HEAD：仅发送命名资源中的http首部
    • PUT：将客户端的数据存储到一个服务器支援中去
    • TRACE:对报文进行追踪
    • OPTIONS：决定可以从服务器上可以执行哪些方法

    人们眼中的GET 和POST的区别：

    1. URL长度的限制，浏览器对URL有大小限制，chrome 8K，firefox 7K，IE 2K
    2. 资源大小限制：GET方法限制大小，因为第一条的关系，而GET又是将数据直接拼接在URL后面，所以GET有大小限制；POST理论上没有大小限制
    3. 功能不同，GET主要是用来从服务器拉取数据，而POST主要是将数据发送到服务器
    4. 相对性安全；GET发送给服务器的数据可以被看到，而POST将数据放在请求主体中，相相对而言安全一些。

    ###HTTP状态码
    每条HTTP响应报文返回时都会携带一个状态码。状态码是一个三位数字的代码，告知客户端请求是否成功或者是否需要采取其他操作。
    几种常见的状态码：

    1. 200 OK 文档正确返回
    2. 301 Redirect 永久重定向。一直从其他地方获取资源
    3. 302 Redirect 临时重定向，临时从其他地方获取资源
    4. 303 see other 将客服端重定向到一个负载不大的服务器
    5. 404 Not Found 无法找到资源
    6. 500 Internal Server Error 服务器错误

    伴随着每个数字状态码，http还会发送一条解释性的文本。
